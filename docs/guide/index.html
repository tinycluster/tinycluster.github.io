Quick Start

Plataform#

The Tiny Cluster has been tested only for the Linux Ubuntu 20.04 operating system. Feel free to test on other platforms as needed. Don't forget to let us know if it worked for you.

  • Linux Ubuntu 20.04

Requeriments#

For you to use the Tiny Cluster make sure that you have installed the following requirements:

  • Vagrant
  • Virtual Box

Quick start#

note

Quick start under construction! Go to Custom start :)

Create a local cluster with the default settings. To customize your installation look at Custom start.

curl -o- [url/install.sh] | bash -x

Custom start#

Create project

mkdir k8s-cluster && cd k8s-cluster

Download file Vagrantfile

wget Vagrantfile

Download file config.yaml

wget config.yaml

Check files

vagrant validate
Output
Key file exists
Vagrantfile validated successfully.

Config cluster

Accessing the config.yaml and Vagrantfile files you can change the settings according to your needs. Operating system you want to install, name of nodes, ip, type node (Master or Worker), amount of cpus, amount of ram, pre-installed software, among other settings.

Build Cluster

vagrant up

Access Cluster#

important

When requesting the password, inform: tinycluster

Acess machine node master

ssh ubuntu@192.168.31.10

Acess machine node worker

ssh ubuntu@192.168.31.11

inform the corresponding IP to access other nodes.

Commands#

Just a few commands that may be useful to you.

Vagrant#

Start cluster.

vagrant up

Clean up the VMs.

vagrant destroy && rm -rf .vagrant

Suspend the current state of VMs.

vagrant suspend

Resume the last state of VMs.

vagrant resume
note

every time you resume the VMs you will find that the machine time is still at you last time you suspended it. So consider to halt the VMs and restart them.
